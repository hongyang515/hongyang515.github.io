React技术之路

摘要: React起源于facebook的内部项目，因对市场上所有JavaScript MVC 框架，都不满意，决定自己设计一套，用来架设Instagram 的网站，并在13年的5月开源，并且一路高歌，目前已经接近七万星，可见其影响力。同时，React的诸多优点，更加满足前端日益发展的需求。

正文:

15年的9月，第一次学习react，并尝试将react运用到公司的eva管理后台项目，被react的设计思想给深深吸引，与其说react是一个前端实现框架，不如说是前端领域的一个新的创新，新的思想：

  1. 相比其他框架，它操作DOM的方式不同，引入虚拟DOM概念，在UI的渲染过程中，React通过虚拟DOM来实现页面真实DOM的局部刷新，提高的WEB的性能。
  2. 单一的数据流思想，让事情变得一目了然。从flux 到redux， 使数据管理更佳方便。
  3. React注重组件式开发，更佳模块化。每个组件都可以独立开发和测试，并能够复用，提高代码的可维护性。
  4. React在生成界面时，与传统web不同，不使用Handlebars或Mustache等模板引擎来处理视图层，是纯粹js编程。

近一年来，react技术在公司前端的普及，项目中的架构已经趋于统一，接下来介绍下react的架构(并不统一):


接下来对每个文件进行逐个介绍：

package.json

对于每个前端开发者来说，package.json是再熟悉不过的文件了，在任何一个项目中都会用到，我们在项目中遇到的所有依赖库都是通过该文件，npm来安装的，package.json主要由三部分组成：

  1. script: 执行脚本，比如文件压缩，启动服务器等操作。
  2. devDependencies： 只用于开发阶段完成集成测试等功能模块依赖。
  3. dependencies: 产品环境依赖库。
    如需更详细了解，请查看相关API。
webpack.config.js

webpack.config.js 是webpack的配置文件，用来告诉webpack是怎么样进行资源打包等相关操作。

dist

该文件夹存放的是经过打包后的代码。

pages

pages存放的是业务模块的组件，文件夹里的index.js是页面的入口，同时配置各个页面的路由。

import Goods from 'react-proxy?name=goods!./goods'
import Good from 'react-proxy?name=good!./good'
const routes = (history) => (
    <Router history={history}>
        <Route path="/" component={Goods} />
        <Route path="/good" component={Good} />
    </Router>
)

上面是index页面的部分代码，包含两个组件，Goods和Good，即商品列表和商品详情，每个组件展示一个页面。

如下图所示，是pages文件夹的具体内容：

根据index路由配置，对应的是每个组件的文件夹，每个文件夹里面存放一个index.js 和style.scss，index里面写的是业务js代码，而scss里面写的是页面的样式。

components

components里面存放的是公共的组件，比如在项目中很多页面都会有footer，而这个footer组件就可以写在这里，这样在每个页面需要时，就可以引入这个组件，提高代码的复用。

assets 和 scss

这两个文件夹存放的是共用的样式，如果需要全局样式，可以写在scss的文件夹的文件里面，而assets里面可以放字体文件信息以及图片信息。

app

这个是项目中比较重要的文件夹，数据的存储和操作都在这里。如图：

  1. actions: 在处理业务逻辑时，在交互过程中会出发很多actions，比如增删改除，不同的业务场景会有不同的操作，在react项目中，数据在state中进行管理，修改state的方式就是通过处发action，即action 用来描述“发生了什么”。

  2. reducer: 通过react-redux管理state，就需要reducer，它是实现(state, action) => newState的纯函数，也就是真正处理state的地方。在reducer里可以通过action对state中的数据进行处理。

  3. selector: 通过selector可以对state的数据进行筛选和组合，将组合后的数据返回给页面中，这样组件从在props中拿到自己需要的数据。

  4. store: Redux 应用只有一个单一的 store。它就是用来维持应用所有的state树的一个对象。

  5. constants: 改文件中纪录页面所有的action，这样能够使项目中所有的action一目了然，并且能够很好避免action重名问题。

utils

该文件夹存放的是项目中所有基础方法。

随着前端框架的广泛应用，React， Angular, Vue备受各个前端团队欢迎，不同的框架，不同的团队拥有不同的项目架构，毕竟思想是独立的😄 。一个好的架构能够很大提高团队的开发效率，以上是本人的经历，以此共勉。
